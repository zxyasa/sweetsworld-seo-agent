## Purpose

The product card is the primary transactional unit on SweetsWorld pages.
It presents a single product with image, name, brief descriptor, and a link to purchase.
Cards are always rendered in a CSS Grid, never in isolation.

---

## Canonical HTML Template

```html
<div class="sweets-product-card" style="
  background: #ffffff;
  border: 1px solid #e7edf3;
  border-radius: 8px;
  padding: 16px;
  display: flex;
  flex-direction: column;
  gap: 8px;
  box-sizing: border-box;
">
  <a href="{{product_url}}" style="display:block; text-decoration:none;">
    <img
      src="{{product_image_url}}"
      alt="{{product_name}}"
      loading="lazy"
      style="
        width: 100%;
        aspect-ratio: 1 / 1;
        object-fit: cover;
        border-radius: 6px;
        display: block;
      "
    >
  </a>
  <h3 style="
    color: #4d5d6c;
    font-size: 1.05rem;
    font-weight: 600;
    margin: 0;
    line-height: 1.4;
  ">{{product_name}}</h3>
  <p style="
    color: #666666;
    font-size: 0.875rem;
    margin: 0;
    line-height: 1.5;
  ">{{product_short_desc}}</p>
  <a href="{{product_url}}" style="
    color: #6bb6d9;
    font-weight: 600;
    text-decoration: none;
    font-size: 0.9rem;
    margin-top: auto;
  ">Shop Now →</a>
</div>
```

## Grid Wrapper Template

Cards are always wrapped in a grid container. Do not render cards without the grid wrapper.

```html
<section class="sweets-products-section" style="margin-bottom: 40px;">
  <h2 style="
    color: #3b4b5c;
    font-size: 1.5rem;
    font-weight: 700;
    margin-bottom: 20px;
    line-height: 1.2;
  ">{{section_heading}}</h2>
  <div class="sweets-product-grid" style="
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 16px;
  ">
    <!-- product cards here -->
  </div>
</section>
```

## Template Variables

| Variable | Source | Notes |
|----------|--------|-------|
| `{{product_name}}` | `products.json` → `product_name` | Required. Truncate at 60 chars if longer. |
| `{{product_url}}` | `products.json` → `url` | Required. Must be a sweetsworld.com.au URL. |
| `{{product_image_url}}` | `products.json` → `image_url` | Required. If null, use placeholder. |
| `{{product_short_desc}}` | Generated by Claude | 1 sentence, 10–25 words. Focus on flavour/occasion. |
| `{{section_heading}}` | Generated per page | Pattern: "Shop [Keyword] at SweetsWorld" |

## Rules

### Must
- Include `loading="lazy"` on every `<img>` tag.
- Use `aspect-ratio: 1 / 1` on the image to prevent layout shift.
- Wrap the image in an `<a>` linking to the product URL.
- Keep card background `#ffffff` (color_card_bg).
- Keep card border `1px solid #e7edf3` (color_card_border).
- Use `border-radius: 8px` on the card, `6px` on the image.
- Use `margin-top: auto` on the "Shop Now" link to push it to the bottom of the card.
- Product name is `<h3>` (H3) — never H2 (H2 is reserved for section headings).

### Must Not
- Do not add product pricing inside the card (prices change and must be live).
- Do not add `<button>` elements — use `<a>` links only.
- Do not use `<table>` for grid layout.
- Do not add `box-shadow` on cards — border only.
- Do not add `target="_blank"` on product links (internal links stay in same tab).
- Do not add a second CTA link per card.
- Do not use `<figure>` or `<figcaption>` — use the template above.

### Conditional
- If `image_url` is null or empty: set `src` to `/wp-content/themes/flatsome/assets/img/placeholder.png` and set `alt` to "{{product_name}} — SweetsWorld Australia".
- If `product_short_desc` is not generated: leave the `<p>` out rather than showing an empty paragraph.
- If the product is out of stock (`is_in_stock: false`): still include the card but change link text to "View Product" instead of "Shop Now →".

## Responsive Behaviour

| Breakpoint | Grid Columns | Card Padding | Image |
|------------|-------------|--------------|-------|
| Desktop (>1024px) | 3 | 16px | aspect-ratio 1:1 |
| Tablet (768–1024px) | 2 | 12px | aspect-ratio 1:1 |
| Mobile (<768px) | 1 | 12px | aspect-ratio 1:1 |

For inline styles (which cannot use media queries), the grid defaults to 3 columns.
Flatsome's responsive system will handle column reduction on smaller screens via its grid framework.
If you need explicit mobile support in the HTML, add a `<style>` comment block as a hint to the theme:

```html
<!-- responsive: grid 3col → 2col@tablet → 1col@mobile -->
```

## Anti-patterns (seen in previously generated pages — do not repeat)

- Using `<ul>/<li>` to list products (creates unordered list appearance instead of card grid).
- Putting pricing text (`$X.XX`) in card descriptions (prices go stale).
- Using `color: #333` or `color: black` instead of `color_text_secondary`.
- Adding `font-family: Arial` inline (conflicts with Flatsome theme fonts).
- Nesting cards inside `<table>` cells.
- Using `<br>` for spacing inside cards (use padding/gap instead).
